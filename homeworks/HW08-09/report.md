# HW08-09 – PyTorch MLP: регуляризация и оптимизация обучения

## 1. Кратко: что сделано

Датасет EMNIST (B) был выбран по причинам сбалансированность классов и доступности.
Часть A (регуляризация):
Сравнивались три подхода к борьбе с переобучением на архитектуре MLP с двумя скрытыми слоями (256 и 128 нейронов):

E1 (base): базовая модель без регуляризации (только линейные слои + ReLU)

E2 (Dropout): добавлен Dropout (p=0.3) после каждого скрытого слоя

E3 (BatchNorm): добавлена Batch Normalization между линейным слоем и активацией

E4 (Early Stopping): лучшая конфигурация из E2/E3 с ранней остановкой (patience=4)

Часть B (оптимизация):
На фиксированной архитектуре (лучшей из части A) исследовалось влияние гиперпараметров оптимизации:

O1 (LR слишком большой): Adam с learning rate = 0.1 (на порядки выше оптимального)

O2 (LR слишком маленький): Adam с learning rate = 1e-5 (на порядки ниже оптимального)

O3 (SGD + momentum + weight decay): SGD с momentum=0.9, weight decay=1e-4 и learning rate=0.01

- Loss: CrossEntropyLoss
- Базовый Optimizer (для части A): Adam с learning rate = 0.001
- Batch size: 64
- Epochs (макс): E1-E3: 15, E4: 30 эпох (с ранней остановкой), O1-O2: 8 эпох, O3: 12
- EarlyStopping: (patience=4, metric=val_accuracy)

## 5. Часть A (S08): регуляризация (E1-E4)

- E1 (base): MLP с тремя скрытыми слоями (512→256→128), активация ReLU, без Dropout и BatchNorm
- E2 (Dropout): как E1 + Dropout (p=0.3) после каждого скрытого слоя
- E3 (BatchNorm): как E1 + BatchNorm между линейным слоем и активацией
- E4 (EarlyStopping): лучшая конфигурация из E2/E3 + EarlyStopping (patience=4)

## 6. Часть B (S09): LR, оптимизаторы, weight decay (O1-O3)

- O1 (LR слишком большой): Adam, lr = 0.1 (на 2 порядка выше оптимального)
- O2 (LR слишком маленький): Adam, lr = 1e-5 (на 2 порядка ниже оптимального)
- O3 (SGD+momentum+weight decay): SGD, lr = 0.01, momentum = 0.9, weight_decay = 1e-4

## 8. Анализ

Переобучение. В E1 (без регуляризации) после 5-6 эпох train loss падает, а val loss растёт — явное переобучение. Dropout (E2) немного сократил разрыв, но BatchNorm (E3) оказался эффективнее: train и val loss идут синхронно, переобучения нет, val accuracy 85.3%.

EarlyStopping. В E4 остановка произошла на 14 эпохе (patience=4), когда val accuracy перестала расти. Это предотвратило лишние итерации и сохранило лучшую модель с test accuracy 92.5%.

Экстремальные LR. O1 (0.1): loss скачет, нет стабилизации, val accuracy 79.5% — слишком большой шаг. O2 (1e-5): loss снижается крайне медленно, accuracy 71.4% — обучение почти не идёт.

SGD+momentum+wd. O3 (SGD, lr=0.01, momentum=0.9, wd=1e-4) достиг 84.9% val accuracy, близко к Adam (85.3%). Momentum ускоряет сходимость, weight decay добавляет регуляризацию — разрыв train/val минимален.

Выбор лучшей модели. E3 с BatchNorm оптимален для EMNIST: 47 классов требуют стабильных признаков, BatchNorm улучшает обобщение, Dropout в данном случае оказался менее эффективным (82.1% против 85.3%).
